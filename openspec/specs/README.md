# OpenSpec — Active Specs

This directory holds the canonical, post-archive specifications for the
pruebaMinimax project (Pico y Placa Bucaramanga).

Specs are added here by the `sdd-archive` phase after a change is verified.
Until the first change is archived, this directory is intentionally empty.

## Conventions

- One file per capability (e.g. `plate-restriction-rule.md`, `scraper-pipeline.md`).
- Each spec uses `## Purpose` plus `## Requirements` with `### Scenario:` blocks.
- Scenarios follow Given/When/Then (gherkin style).
- No implementation detail lives here; only behavior and contracts.
